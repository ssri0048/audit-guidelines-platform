ที่มาของแต่ละวิธีการตรวจ: 📖 อ่านจากแหล่งจริง 🧭 อิงแบบแผนจากคลังตัวอย่างผู้ตรวจจริง ⚗️ สังเคราะห์เชิงวิชาชีพ (ยึดมาตรฐานที่ verify แล้ว)

🛡️ ธรรมาภิบาลความรู้ของระบบนี้

ทุกหัวข้อผ่านกระบวนการ re-verify จากแหล่งทางการ (มีลิงก์+ข้อความอ้างอิงจริงทุกรายการ) → ตรวจด้วย quality gates อัตโนมัติ (schema, เวอร์ชันมาตรฐานปัจจุบัน, ความใช้ได้ในไทย, กันปีอนาคตปลอม) → มนุษย์อนุมัติผ่าน Pull Request เท่านั้น หน้าเว็บนี้แก้ไขข้อมูลไม่ได้ — การเพิ่ม/แก้ความรู้ทำผ่าน GitHub

➕ ขอเพิ่มหัวข้อใหม่ (สั่งระบบประมวลผลจากหน้านี้) 🚩 แจ้งข้อมูลคลาดเคลื่อน 📂 ดูฐานความรู้บน GitHub

🤖 สั่งระบบสร้างร่างหัวข้อใหม่

เว็บนี้ยังอ่านอย่างเดียว — คำสั่งนี้แค่สตาร์ท pipeline หลังบ้าน ผลลัพธ์คือ draft PR ที่ต้องผ่านเกต + คนอนุมัติเสมอ

สร้าง token: GitHub → Settings → Developer settings → Fine-grained tokens → เลือกเฉพาะ repo นี้ + Permission เดียว: Actions: Read and write — token นี้สั่งงาน/ดูสถานะได้เท่านั้น เขียนความรู้ไม่ได้
🛡️ ธรรมาภิบาลไม่เปลี่ยน: ระบบจะทำ Reading Manifest (อ้างได้เฉพาะมาตรฐานที่มีทะเบียน) → ร่าง L3 → เปิด PR — ความรู้เข้าระบบจริงต่อเมื่อผ่าน CI และคุณกด merge เท่านั้น
